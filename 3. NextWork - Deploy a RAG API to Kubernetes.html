Introducing Today's Project!

In this project, I will deploy Kubernetes. I'm doing this project to learn about Pods, Services and Kubercli. Kubernetes will help me to restart container if it crashs or error.

Key services and concepts

Key concepts I learnt include Pods, Deployments, and Services. Kubernetes provides a platform to run, manage, and network containerized applications reliably. Deployments manage the lifecycle of Pods, ensuring the desired number of replicas are running and handling restarts. Services route traffic to Pods using label selectors and provide stable access to applications even when Pods are recreated.

Challenges and wins

This project took me approximately one days to complete. The most challenging part was debugging container networking and service communication between the API and Ollama in Kubernetes. It was most rewarding to see the application successfully respond to requests once all the components were correctly connected and running in the cluster.

Why I did this project

I did this project because I wanted to learn Kubernetes and other RAG apps. One thing I'll apply from this is Kubernetes to prevent production down-time

Creating a Service

In this step, I'm creating a Kubernetes Service by creating the service.yaml file. I need a Service to provide a stable network endpoint for my Pods

What does the service.yaml file do?

The service.yaml file tells Kubernetes to assign rag-app-service on Port 8000. The selector finds Pods by matching the port and targetPort.

What kubectl commands did you run to create the service?

I applied my Service file by running kubectl apply -f service.yaml. I then verified that the Service was created by running kubectl get services

Accessing My API Through Kubernetes

In this step, I'm testing my API

How I accessed my API

I tested my API by sending HTTP requests using curl to the NodePort exposed by Kubernetes. The response showed that the API was running correctly and able to generate answers using the Ollama model. This confirms that the Service correctly routed traffic to the Pod and that the application was fully functional inside the cluster. The main difference between Docker and Kubernetes deployment is that Kubernetes adds a networking and service layer to manage Pods, scaling, and communication instead of running a single container directly.

Request flow through Kubernetes

The request flow went from my computer to the Minikube node using the node‚Äôs IP address and a NodePort exposed by Kubernetes. When I sent the HTTP request (using curl), it was addressed to: http://<minikube-ip>:<nodePort> This traffic first reached the Kubernetes Node (Minikube VM). The NodePort Service was configured to listen on that port on every node, so Kubernetes accepted the incoming connection at the node level. The Service then acted as a stable entry point and routed the traffic to one of the Pods matching its label selector (in this case, the RAG API Pod). Internally, the Service used kube-proxy to forward the request to the Pod‚Äôs IP and the container port where the FastAPI application was listening. Inside the Pod, the request reached the FastAPI server (uvicorn) running on port 8000. The /query endpoint handled the request, queried ChromaDB, and then made an internal call to the Ollama Service (ollama:11434) to generate the response.

Testing Self-Healing

In this project extension, I'm demonstrating that Pod are automatically generated as old pod got deleted. Self-healing is important because it prevents production app to stop working due to memory leak, container crashs, etc...

What did you observe when you deleted the pod?

When I deleted the pod, I saw the status change from Running to Terminating to Completed. A new pod was created at the same time as well because the other pod was deleted.

How the Service routed traffic to the new pod

The Service automatically restart. Without Kubernetes this would have become disastrous that i have to manual restart the docker container by myself. Self-healing is critical in production because it ensures persistent working apps.
