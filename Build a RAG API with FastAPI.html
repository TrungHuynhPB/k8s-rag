How do I port Ollama on port 11434 using Docker ?

In this project, the approach is to run

Ollama
directly on your host machine (your computer) and then configure your
RAG API
 container to connect to it. This keeps the setup simpler for learning purposes.

You'll find the details on how to ensure your containerized

RAG API
can reach
Ollama
 running on port 11434 on your host machine in the project. Specifically, you need to update your app.py file to use host.docker.internal:11434.

You can find this information in the Step #3: Containerize Your API section, under the troubleshooting for "Internal Server Error" or "Ollama connection error" when testing your containerized API. It explains how to modify your app.py to create an

Ollama
client that knows where to find
Ollama
 on your host.

While running

Ollama
in its own container is a common production pattern, for this project, we're connecting to it from the host.

API query breakdown

I queried my API by running the command curl -X POST "http://127.0.0.1:8000/query" -G --data-urlencode "q=What is Kubernetes ?" The command uses the POST method, which means it will transfer the string from my input cli to variable q and then use llm to respond
